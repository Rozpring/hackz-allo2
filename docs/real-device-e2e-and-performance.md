# 実機 E2E・性能計測チェックリスト（#35 / task 9.3）

シミュレータでは ARKit ワールドトラッキング・Vision 手検出・RealityKit 物理の実挙動を検証できないため、
本チェックリストに沿って **実機**（ARKit 対応 iPhone, A12 Bionic 以降目安 / iOS 16+）で確認する。
ユニット/統合テストは main で 87 件緑（`xcodebuild test`）。本書はそこで担保できない「実機動線・体感・性能」を対象とする。

対象コンポーネント（main 実装済み）:
`ARSceneController` / `VisionHandProvider` / `HandSwingDetector` / `PowerCalculator` /
`ShockwaveSystem` / `PhysicsSettleObserver` / `MatchEvaluator` / `GameStateManager` /
`GameSession` / `GameEngine` / `HUDView` / `ResultView` / `FeedbackController`。

## 0. 事前準備

- [ ] `xcodegen generate` 後、実機ターゲットで署名（個人 Team で可）してインストール
- [ ] 効果音アセット（`punch.wav` / `flip.wav` / `pair.wav`）をアプリバンドルへ同梱（未同梱でも触覚・進行は動作）
- [ ] テーブル等の水平面と、片手で叩ける環境を用意

## 1. 主要動線 E2E（要件 R1〜R8）

- [ ] **起動 → 権限要求**: 初回起動でカメラ許可ダイアログが出る（R1-1）
- [ ] **権限拒否 → 導線**: 拒否すると機能不可メッセージ＋「設定を開く」が出て、盤面へ進まない（R1-2）
- [ ] **許可 → トラッキング開始**: 背面カメラ映像が表示され、水平面検出が走る（R1-3）
- [ ] **平面ガイド**: 平面未検出時はガイド、十分な広さ検出で配置可能提示（R1-4/R1-5、`PlacementGuidance`）
- [ ] **盤面配置**: 「盤面を配置して開始」で 8 ペア=16 枚が格子・全伏せで AR 配置（R2-2/R2-3）
- [ ] **アンカー固定**: 端末を動かしても盤面が現実空間に固定され続ける（R2-5/R10-3）
- [ ] **台パン検出**: 空いた手で叩くと `VisionHandProvider`→`HandSwingDetector` で台パン成立（R3/R4-1）
- [ ] **威力→衝撃波**: 叩く速さで威力が変化し、半径内カードがめくれる（R4-2/R5-1）
- [ ] **静止→表裏確定**: カードが落ち着くと表/伏せが確定（R5-6/`PhysicsSettleObserver`）
- [ ] **ペア自動回収・加点**: 表の同一ランク2枚が自動成立・回収されスコア加算（R6）
- [ ] **コンボ**: 1回の台パンで複数ペア成立時に倍率が上がる（R6-4、HUD の COMBO 表示）
- [ ] **全クリア**: 残ペア0で結果画面（クリア＝経過時間・スコア）（R8-2）
- [ ] **タイムアップ**: 制限時間0で結果画面（残ペア・スコア）（R8-3）
- [ ] **リトライ**: 「もう一度あそぶ」で初期状態に戻り再プレイできる（R8-4）

## 2. 物理の非決定論・後続台パン（要件 R5/R7）

- [ ] 強い台パンほど影響半径が広く、めくれる枚数が増える（R5-2）
- [ ] 同じ強さでも結果がばらつく（浮くだけ/半回転/一回転/複数回転）（R5-3/R5-5）
- [ ] 表になっているカードが後続の台パンで再び伏せに戻り得る（R7-3、疑似スリープ復帰）
- [ ] カードが盤外へこぼれない（床＋外周4壁コライダー）（R5-8）

## 3. 手検出・フィードバック（要件 R3/R9/R5-7）

- [ ] 手をかざすと代表点が安定して流れ、フレーム外に出すと発行が止まる（R3-4）
- [ ] 手未検出時に HUD の補助ガイド「手をカメラに写してください」が出る（R9-4、`isHandDetected`）
- [ ] HUD が残時間・スコア・威力ゲージ・コンボを更新し、AR 盤面の視認を妨げない（R9）
- [ ] 台パン/ペア成立で触覚（CoreHaptics）＋効果音が再生される（R5-7、`FeedbackController`）

## 4. 性能計測（要件 R10-1、Instruments）

- [ ] ARKit＋Vision手検出＋16枚物理の同時稼働で滑らかな実効FPS（目標: 体感で滑らかにプレイ可能）
- [ ] 手検出レイテンシと間引き挙動: `VisionHandProvider` の in-flight ガードでキューが詰まらない
- [ ] 連続台パン時のメモリ健全性: `capturedImage`(CVPixelBuffer) が適切に解放されプールが枯渇しない
- [ ] 静止後の負荷低下: 疑似スリープ（`.static` 化）で可動カード減少時に負荷が下がる

## 5. 異常系・堅牢性（要件 R10-3/R10-4）

- [ ] トラッキング品質低下（`limited`）時にガイドが出て、復帰後に継続できる
- [ ] 物理が静止しないケースでもウォッチドッグで確定し、クラッシュせず進行する（`PhysicsSettleObserver` watchdog）
- [ ] Vision 推論の例外/タイムアウト時に当該フレームを破棄し次フレームへ（クラッシュしない）
